Учебный проект с использованием микросервисной архитектуры Spring Cloud
Данный проект представляет собой простую имитацию работы банковской системы
Проект состоит из 7-ми микросервисов:
- gateway-service (обрабатывает входящие http запросы, перенаправляет их в соответствующий сервис);
- discovery-service (в данном сервисе регистрируются все микросервисы). Также работает в связке с gateway-service;
- config-service (запускается первым, в данном сервисе все остальные сервисы получают свои файлы конфигурации);
- account=service (связан со своей базой данной, обеспечивает выполнение различных CRUD операций с сущностью Account);
- deposit-service (связан со своей базой данных и account-service). Данный сервис служит для проведения операции
пополнения счета того или иного аккаунта;
- rabbitMQ (брокер сообщений, получает информацию о пополнении счета и передает ее в notification-service).
Можно запустить через Docker;
- notification-service (служит для отправки сообщения на электронную почту через SMTP владельцу аккаунта на чьем счете
произошло пополнение);

Также в планах добавить transaction-service для списания средств со счета и transfer-service для перевода средств между
счетами

Также прилагается схема (spring-cloud-microservice-schema.drawio), которая визуально демострирует работу приложения
и взаимосвязи между сервисами.

Открыть можно через сайт https://www.drawio.com
